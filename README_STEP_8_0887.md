# 0.8.8.7-native-bridge-return-freeze

基线：0.8.8.6-extra-wave-return-freeze。

修复目标：第 8 波 -> 第 9 波桥接阶段，玩家离场/回归时不能因为袭击者暂时卸载或扫描为空而误判第 8 波已清完，导致 UI 跳到第 9 波并刷新新型袭击者。

核心改动：
- 将 0.8.8.6 的“额外波无玩家立即冻结”扩展到“原版最终波桥接窗口”。
- 第 8 波之后、额外波尚未真正生成之前，如果没有有效参战玩家，冻结桥接状态机。
- 玩家回归后增加 60 tick 短暂加载宽限，避免回归第一瞬间扫描为空就推进波次。
- nativeRaidFinishedObserved 路径和 pre-victory 路径都要求玩家在场且加载宽限结束后，才允许用 nearbyRaiders <= 0 判定桥接。
- 不传送、不清怪、不失败、不改原版 Raid 状态。
