# 第九步热修：波次计划 / HUD / 桥接全面同步

## 问题

用户反馈：很多难度与不祥等级下，实际刷新波次和 UI 显示波次对接不上。

## 根因

旧逻辑中同时存在三套波次来源：

1. `RaidWaveExpansionPlan` 计划表。
2. `Raid.numGroups` 原版字段估算。
3. `groupsSpawned` 实际已生成波数。

其中 `Raid.numGroups` 和理论不祥额外波曾被用于抬高 HUD 目标或桥接目标，导致 HUD 和实际刷新不一致。

## 修复策略

本版改为：

- 目标总波数只认 `RaidWaveExpansionPlan`。
- `groupsSpawned` 只作为进度证据。
- `Raid.numGroups` 只作为原版状态观察，不作为 HUD 总目标。
- 原生安全上限仍为 7，超过 7 的目标统一由自定义桥接补齐。
- 进入自定义波后锁定计划目标，避免波数漂移。

## 计划表

- 简单：不祥 1/2/3/4/5 = 3/3/4/4/5。
- 普通：不祥 1/2/3/4/5 = 5/6/6/7/8。
- 困难：不祥 1/2/3/4/5 = 8/8/9/10/11。

## 未改动

- 不改胜利结算奖励。
- 不改 JSON 奖池。
- 不改战场扫荡兑换。
- 不改村庄恩情。
- 不改 Raids Enhanced 魔像回滚和掉落物清理。
- 不改战备令牌。
