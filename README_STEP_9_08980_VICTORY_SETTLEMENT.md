# 第九步 0.8.9.8.0：袭击胜利结算 / 战场扫荡 / 村庄恩情 Alpha

## 本版定位

基于 `0.8.9.7.20-golem-drop-cleanup-query-hotfix` 稳定线，新增第一版战后结算闭环。

## 已实现

- 战报从两档调整为三档：完胜 / 胜利 / 惨胜。
- 胜利结算只执行一次，并写入 `victory_settlement_history.properties` 防止重启后重复发奖。
- 结算接入 `VillageSecurityController.complete(...)` 胜利分支，不重写原版 Raid，也不改第 9～11 波状态机。
- 保留额外波判断基础：结算由现有完成点触发，因此有额外波的袭击必须等额外波链完成后才会进入结算。
- 新增 JSON 奖励：
  - `perfect_victory.json`
  - `victory.json`
  - `costly_victory.json`
- 新增经验奖励 JSON：困难不祥 5 胜利基础 1507 XP，并按评级倍率调整。
- 新增战场扫荡：默认回收弩、铁斧、鞍、灾厄旗帜；不死图腾默认关闭回收。
- 新增村庄恩情第一版：玩家 UUID + 维度 + 村庄中心绑定；记录恩情点数与等级；进村提示；战后增加恩情。
- 已按用户要求：主城防卫加成暂不实现，不提高安防铁傀儡部署数量或抗性时间。

## 未改动

- 未改第 8 波桥接。
- 未改第 9～11 波额外波状态机。

## 风险说明

当前环境没有 Gradle Wrapper 和 Gradle 命令，因此本 JAR 使用“旧稳定 JAR 解包 + 编译改动类覆盖”的方式生成；编译时使用本地兼容桩校验 Java 语法与类输出，未在真实游戏客户端内实测。若启动崩溃，请发最新崩溃报告，我会按 NeoForge 21.1.234 运行时签名继续热修。
