# 安装说明：raid_enhancement_patch-0.8.9.8.3-wave-plan-ui-sync-hotfix

## 版本定位

本版是 0.8.9.8.2 的波次系统全面检修热修版。

重点修复：多个难度 / 不祥等级下，HUD 显示波次、原版实际刷新波次、自定义桥接波次、胜利结算等待条件不完全同步的问题。

## 安装

1. 删除 mods 文件夹内旧版本 `raid_enhancement_patch-*.jar`。
2. 只放入：`raid_enhancement_patch-0.8.9.8.3-wave-plan-ui-sync-hotfix.jar`。
3. 不要多个版本共存。

## 继承内容

本版继承：

- 0.8.9.8.0 胜利结算 / JSON 奖池 / 战场扫荡 / 村庄恩情第一版。
- 0.8.9.8.1 LevelTickEvent 进世界崩溃热修。
- 0.8.9.8.2 困难 + 不祥 1 第 8 波桥接修复。

## 本版新增修复

- 目标总波数统一只认 `RaidWaveExpansionPlan` 计划表。
- HUD 总波数不再用 `Raid.numGroups` 或理论不祥额外波估算抬高。
- 自定义桥接基准只认“实际已生成的原生波数 + 原生安全上限”。
- 进入自定义桥接后锁定计划目标，避免 HUD 跳到 9/10/11 之外。
- 原生 `numGroups` 在安全范围内也会尽量对齐计划表，避免实际原版波次多于 HUD。
- 已生成波数只作为进度证据，不再反推普通显示目标。

## 测试建议

建议新开袭击测试，不要接着旧异常袭击测试：

- 简单 + 不祥 1～5：3 / 3 / 4 / 4 / 5。
- 普通 + 不祥 1～5：5 / 6 / 6 / 7 / 8。
- 困难 + 不祥 1～5：8 / 8 / 9 / 10 / 11。

重点看：HUD 显示、实际波次、自定义桥接、胜利结算是否一致。
