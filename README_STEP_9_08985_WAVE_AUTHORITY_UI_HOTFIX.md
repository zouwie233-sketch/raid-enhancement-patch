# 第九步 0.8.9.8.5：波次权威层与 UI 兜底热修

## 修复目标

本版把目标总波数、原版负责波数、自定义额外波数量统一到 `RaidWaveAuthority`。

## 波次结构

- 简单：3 / 3 / 4 / 4 / 5
- 普通：5 / 6 / 6 / 7 / 8
- 困难：8 / 8 / 9 / 10 / 11
- 原版 Raid 只负责 1～8 波。
- 自定义额外波只允许 9～11 波。

## 关键修复

1. `groupsSpawned` 不再抬高 HUD 总波数。
2. `Raid.numGroups` 写入、HUD、桥接、胜利抑制使用同一权威表。
3. `armExtraWaves` 增加硬门槛：目标波数必须大于 8。
4. 8 波以内袭击会清空旧的 armed/bridge/custom 状态。
5. 生命周期恢复时，新袭击开始阶段不继承旧快照。
6. 客户端 HUD 获取 `GuiGraphics` 改为反射兼容。
7. 增加服务器 Actionbar 兜底显示“袭击波次：第 X / Y 波”，即使客户端覆盖层失效也能看到波次。
